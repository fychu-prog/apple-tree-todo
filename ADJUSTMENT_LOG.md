# 🍎 蘋果待辦樹 (Apple Todo Tree) - 調整紀錄

這份文件詳列了專案開發過程中的所有核心修正與參數微調，作為後續維護與 AI 協作的指導手冊。在進行任何代碼修改前，請務必先閱讀此文件。

---

## 📅 最後更新日期：2026-04-09

### 1. 物理引擎 (Matter.js) 參數定錨

- **蘋果規格**：
  - 視覺尺寸：`100px x 106px`。
  - 物理半徑 (`AR`)：`50`。
  - 碰撞與密度：`density: 0.04` (增加重量感，確保蘋果能沉入堆疊中)，`frictionStatic: 1.0` (確保堆疊穩定，不會因微小震動位移)。
- **座標映射**：
  - 籃子物理界限基底為 `320x220` 映射至實體寬度（通常為 `600px` 或 `650px`）。
  - 地面 (`gndCy`) 座標校準必須嚴格對齊籃子底部，目前基於 SVG 視圖比率計算。

### 2. UI/UX 與佈局規範

- **樹木與生長空間**：
  - SVG `viewBox` 已擴展至 `-500 -600 1000 1200`，確保巨大的樹冠不會被切邊。
  - 蘋果生成中心 (`tx, ty`) 定錨於 `(300, 160)`，配合 `rx=240, ry=250` 的圓周隨機算法。
- **輸入介面 (Input Logic)**：
  - 採用 **80% (文字框) : 20% (蘋果按鈕)** 的黃金比例，平衡手機單手操作的舒適度。
- **字體分級**：
  - 標題 `h1`：`1.5rem` (極簡主義)。
  - 蘋果內文字：`1.65rem` (確保手機遠端閱讀清晰)。
  - 清單/布告欄文字：`1.15rem`。

### 3. PWA 與 行動端優化

- **交互行為**：
  - 所有蘋果元素必須帶有 `touch-action: none` 以防止拖拽時觸發瀏覽器頁面滾動。
  - **陀螺儀 (Gyroscope)**：接入 `deviceorientation` 事件，映射角度至 `engine.world.gravity.x/y`。iOS 需要在用戶點擊 `addBtn` 時觸發 `requestPermission`。
- **緩存管理 (Service Worker)**：
  - 採用版本號控制 (`STORAGE_KEY` 與 `sw.js` 緩存名同步)。
  - 開發階段修改 CSS/JS 後，需建議用戶重新整理 3 次以突破強制緩存。

### 4. 邏輯機制

- **承載上限**：樹上最大蘋果數設為 **22 顆**。
- **提示語**：當達到上限時，必須展示充滿正向鼓勵的繁體中文提示，引導用戶採收（完成任務）。

## 🔮 未來展望與增強功能 (Future Enhancements)

以下為根據遊戲化生產力最佳實踐整理出的建議功能，未來可依序實裝：

1. **🚀 搖一搖採收 (Shake to Harvest)**：
   - 利用加速度計監測大幅度震動。
   - 觸發時，讓樹上所有「已標記完成」的蘋果模擬受力震落，增添互動樂趣。

2. **🚦 優先級配色系統 (Apple Color Priorities)**：
   - 允許用戶在輸入任務時選擇顏色（紅/黃/綠）。
   - 紅色：緊急（視覺佔比最顯眼）；綠色：日常；金色：大目標。

3. **🎵 採收音效波 (Auditory Feedback)**：
   - 種植成功：輕微的「波」聲或發芽聲。
   - 採收落下：清脆的木琴聲或金屬籃碰撞聲。

4. **📊 歷史果園畫冊 (Orchard History)**：
   - 紀錄每日採收數量，並以「果園規模」的形式視覺化展示過去一週的產力。

5. **🍂 熟透與腐爛機制 (Urgency System)**：
   - 任務停留過久（如 > 48hr）蘋果顏色變深，提醒用戶及時完成。

## 📅 最後更新日期：2026-04-14 (v31p)

### 5. 響應式佈局 (Responsive Layout)
- **核心目標**：實現跨裝置「零捲動」與「零重疊」的沈浸式體驗。
- **邏輯幾何系統**：
  - 建立 `garden-transform` 容器，作為統一縮放基準。
  - **邏輯解析度**：定錨為 `900x1550` (邏輯寬x高)，確保在長比例手機（如 iPhone 14 Pro Max）上也有足夠的垂直緩衝。
  - **安全邊距**：Header 下方強制預留 `120px` 的 `top` 偏移量，徹底杜絕樹冠與輸入框重疊。
- **縮放基準變革**：
  - 放棄依賴視窗全局高度，改採 `garden-wrapper.clientHeight` 作為動態縮放係數 (`scale`) 的唯一基準。
  - 容器對齊改為 `align-items: flex-start`，確保 scaler 永遠從 Header 底端開始排列，而非在極短螢幕中因居中而溢出至 Header 區域。
- **籃子定位精確化**：
  - 徹底移除負邊距 (`margin-top: -390px` 等)，改採絕對座標。
  - 籃子基底定錨於邏輯座標 `top: 880px`，精確鎖定在樹幹底部與草地交界。

---

> **警告**：修改任何 CSS 定位時，請務必先確認 `main.js` 中的 `logicalH` 參數，此數值決定了整個座標系的「密度」。
